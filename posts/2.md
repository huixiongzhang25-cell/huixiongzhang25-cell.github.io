 ##                     数据库
数据库（Database）是按照数据结构来组织、存储和管理数据的仓库。
每个数据库都有一个或多个不同的 API 用于创建，访问，管理，搜索和复制所保存的数据。
我们也可以将数据存储在文件中，但是在文件中读写数据速度相对较慢。
所以，现在我们使用关系型数据库管理系统（RDBMS）来存储和管理大数据量。所谓的关系型数据库，是建立在关系模型基础上的数据库，借助于集合代数等数学概念和方法来处理数据库中的数据。

## 登录 MySQL

当 MySQL 服务已经运行时, 我们可以通过 MySQL 自带的客户端工具登录到 MySQL 数据库中, 首先打开命令提示符, 输入以下格式的命名:

mysql -h **主机名** -u **用户名** -p
### 参数说明：

| 类型            | 含义                                                            |
| ------------- | ------------------------------------------------------------- |
| -h (host)     | 指定客户端所要登录的 **MySQL 主机名**, 登录本机(localhost 或 127.0.0.1)该参数可以省略; |
| -u (username) | 登录的用户名;                                                       |
| - p(password  | 告诉服务器将会使用一个密码来登录, 如果所要登录的用户名密码为空, 可以忽略此选项。                    |

**如果我们要登录本机的 MySQL 数据库，只需要输入以下命令即可:**
```mysql -u root -p```

按回车确认, 如果安装正确且 MySQL 正在运行, 会得到以下响应:
```
Enter password:
```

- 若密码存在, 输入密码登录, 不存在则直接按回车登录。登录成功后你将会看到 **Welcome to the MySQL monitor... **的提示语。

- 然后命令提示符会一直以 mysql> 加一个闪烁的光标等待命令的输入, 输入 **exit** 或 **quit** 退出登录。

#                        SQL数据类型

### 数值类型

| 类型           | 大小      | 含义            |
| ------------ | ------- | ------------- |
| TINYINT      | 1 Bytes | 极小整型          |
| SAMLLINT     | 2 Bytes | 小整型           |

### 时间类型
| 类型        | 格式                  | 用途                       | 优/劣势                 |
| --------- | ------------------- | ------------------------ | -------------------- |
| DATE      | YYYY-MM-DD          | 存储用户的生日、入职信息、离职信息...     | 不能够绑定日期              |
| DATETIME  | YYYY-MM-DD hh:mm:ss | 用于精确时间点的业务：用户登录时间，支付完成时间 | 固定时间点业务的首选，但占用8字节    |
| TIMESTAMP | YYYY-MM-DD hh:mm:ss | 和DATETIME的业务类型相似         | 最大只能到2038年，会随数据库时区变化 |
| TIME      | HH:MM:SS            | 店铺营业时间、打卡时间、视频时长...      |                      |

### 字符串类型
| 类型       | 大小                    | 用途          |

#                         SQL对字段进行约束

>通过给字段进行约束，让字段的数值只能在约束条件下取值

| 约束词            | 含义                             | 用途                                              |
| -------------- | ------------------------------ | ----------------------------------------------- |
| NOT NULL       | 非空约束：该字段的值不能为NULL(必须填写)        | username VARCHAR(20) NOT NULL(用户名不能为空)          |
| UNIQUE         | 唯一约束：该字段的值在整张表中不能重复(可以有一个NULL) | 唯一性                                             |
| PRIMARY KEY    | 主键约束                           | 用做表的唯一索引，用于快速定位表的关联数据                           |
| FOREIGN KEY    | 外键约束：建立两张表之间的关联                | 用于多表关联，如订单表中的用户id关联到用户表                         |
| CHECK          | 检查约束：限制字段的值必须满足特定条件            | 用户数据合法性判断 age INT CHECK (age BETWEEN 0 AND 120) |
| AUTO INCREMENT | 自增约束：该字段的值会自动增加                | 用于生成连续id,常和主键搭配使用                               |
| DEFAULT        | 默认值约束：插入指定数据时，若未指定值则自动使用默认值    | 避免重复填写，比如用户表中用户状态默认为1(正常)                       |

## 创建数据表

**使用CREATE TABLE <table_name>(字段一，字段二，...)创建数据表：**
```
CREATE TABLE user(
user_id INT AUTO_INCREMENT PRIMARY KEY,
username VARCHAR(50) NOT NULL,
age SMALLINT CHECK(age>0 AND age<120) ,
user_password VARCHAR(25) NOT NULL
);
```


如果发现你不小心给你的数据表命名错了，比如数据表**user** 命名为**uesr**,可以通过以下命令实现**表名的修改：**
```
ALTER TABLE uesr RENAME to user;
```


同样，可以对**字段名也可以进行修改**：
```
ALTER TABLE user rename column `user_password` to `password`;
```


如果想给数据表**新增字段**，可以使用以下命令：
**ALTER TABLE <table_name> ADD CONLUMN**
```
ALTER TABLE user ADD COLUMN `user_status` TINYINT NOT NULL DEFAULT 1 COMMENT "用户状态： 1正常 0禁用";
```

>字段名要用反引号包裹，避免和关键字冲突

同样，可以对**已有的字段**进行修改：
```
ALTER TABLE user MODIFY COLUMN `password` VARCHAR(50) COMMENT "密码";
```

## 查询数据

**查找所有列的数据：**
```
SELECT * FROM table_name
WHERE username="张三";
```

 **查找特定列:**
```
SELECT username,id FROM table_name
WHERE username="万一丁";
```


### 模糊查找

**使用LIKE关键字：**
```
SELECT * FROM table_name
WHERE student_name LIKE '张'
```

### 通配符查找

**%通配符**(表示零个或多个字符)
```
SELECT * FROM table_name
WHERE username LIKE '张%'
```

>这个语句表示匹配已张开头的所有username,比如"张三"，"张一"，"张二"都能匹配到

**\_通配符**(表示单个字符) 
```
SELECT * FROM table_name
WHERE student_name LIKE "_张%"
```

### 合并关键字(UNION)

#### 选取列，并合并

```
SELECT student_name FROM student
UNION
SELECT student_city FROM student
ORDER BYs tudent_city;
```
